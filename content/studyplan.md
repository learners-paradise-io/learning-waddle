# 📅 Interview Study Plans

<div class="section-banner">
  <span class="banner-icon">🎯</span>
  <span class="banner-text">Two structured study plans tailored to your schedule. Choose the 3-day sprint if the interview is soon, or the 7-day plan if you have more time. Follow the plan day by day — each day builds on the previous one.</span>
</div>

---

## ⚡ Plan A: 3-Day Sprint

> Best when you have limited time. Focus on the highest-impact areas.

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">1</span>
    <span class="day-title">ETL & SQL Foundations</span>
    <span class="day-time">⏱️ 4-5 hours</span>
  </div>
  <div class="plan-day-body">

**Morning — ETL Core (2 hours)**
- [ ] Read [ETL Fundamentals](content/learn.md#what-is-etl) — understand Extract, Transform, Load
- [ ] Study [ETL Architecture](content/learn.md#etl-architecture) — know the data flow layers
- [ ] Review [Data Transformation Types](content/learn.md#data-transformation-types)
- [ ] Read [ETL Testing Lifecycle](content/etl_testing.md#etl-testing-lifecycle)

**Afternoon — SQL Core (2-3 hours)**
- [ ] Practice [SQL Basics](content/learn.md#sql-fundamentals) — SELECT, WHERE, GROUP BY, HAVING
- [ ] Master [JOINs](content/learn.md#joins) — especially INNER and LEFT JOIN
- [ ] Study [NULL Handling](content/learn.md#null-handling) — COALESCE, IS NULL, NULL gotchas
- [ ] Practice [Row Count & Missing Record queries](content/sql_practice.md#1-row-count-validation)

**Evening — Quick Test**
- [ ] Do all [ETL flashcards](content/quickrecall.md) — test your recall
- [ ] Review the [Cheat Sheet](content/cheatsheet.md) for quick reference

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">2</span>
    <span class="day-title">Advanced SQL & Interview Strategy</span>
    <span class="day-time">⏱️ 4-5 hours</span>
  </div>
  <div class="plan-day-body">

**Morning — SQL Deep Dive (2 hours)**
- [ ] Study [Subqueries](content/learn.md#subqueries) — correlated and non-correlated
- [ ] Master [Window Functions](content/learn.md#window-functions) — ROW_NUMBER, RANK, LAG
- [ ] Practice [CTEs and Temp Tables](content/learn.md#common-table-expressions-ctes-and-temp-tables)
- [ ] Work through [Duplicate Detection](content/sql_practice.md#3-duplicate-detection) and [Referential Integrity](content/sql_practice.md#7-referential-integrity-checks) queries

**Afternoon — ETL Testing Practice (1.5 hours)**
- [ ] Work through [Scenario Labs](content/scenarios.md) — all 8 scenarios
- [ ] Review [Common ETL Defect Patterns](content/etl_testing.md#common-etl-defect-patterns)

**Evening — Interview Prep (1.5 hours)**
- [ ] Read [Interview Tips](content/tips.md) — STAR framework, model answers, red flags
- [ ] Study your [Resume Guide](content/resume_guide.md) — key talking points for each role
- [ ] Practice saying answers out loud

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">3</span>
    <span class="day-title">Mock Interview & Confidence Building</span>
    <span class="day-time">⏱️ 3-4 hours</span>
  </div>
  <div class="plan-day-body">

**Morning — Full Rehearsal (2 hours)**
- [ ] Take the [Timed Quiz](content/quiz.md) — aim for 80%+
- [ ] Go through [Mock Interview](mock_interview.md) — practice answering each question OUT LOUD
- [ ] Review any questions you struggled with

**Afternoon — Final Review (1-2 hours)**
- [ ] Re-read the [Cheat Sheet](content/cheatsheet.md) — burn it into memory
- [ ] Review the [Mind Map](content/mindmap.md) for visual reinforcement
- [ ] Quick pass through [Flashcards](content/quickrecall.md) — focus on ones you missed
- [ ] Read [Pre-Interview Checklist](content/tips.md#-pre-interview-checklist) and verify everything

**Before Bed**
- [ ] Prepare interview logistics: outfit, location/link, resume copies
- [ ] Get good sleep — a rested mind performs 10x better

  </div>
</div>

## 📚 Plan B: 7-Day Comprehensive

> Best when you have a week. Covers everything with time for practice and repetition.

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">1</span>
    <span class="day-title">ETL Fundamentals</span>
    <span class="day-time">⏱️ 3 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Read [What is ETL?](content/learn.md#what-is-etl) — thoroughly understand all three stages
- [ ] Study [ETL Architecture](content/learn.md#etl-architecture) — data flow layers and tools
- [ ] Learn [Data Transformation Types](content/learn.md#data-transformation-types) — all 7 types
- [ ] Review [ETL vs ELT](content/learn.md#etl-vs-elt) — know the difference
- [ ] Study the [ETL Mind Map](content/mindmap.md) for visual reinforcement
- [ ] Do the ETL section of [Flashcards](content/quickrecall.md)

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">2</span>
    <span class="day-title">SQL Basics + JOINs</span>
    <span class="day-time">⏱️ 3 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Review [SQL Fundamentals](content/learn.md#sql-fundamentals) — SELECT, WHERE, ORDER BY
- [ ] Master [GROUP BY and HAVING](content/learn.md#group-by-and-aggregate-functions)
- [ ] Deep-dive into [JOINs](content/learn.md#joins) — draw diagrams, understand each type
- [ ] Practice [Row Count](content/sql_practice.md#1-row-count-validation) and [Missing Record](content/sql_practice.md#2-finding-missing-records-source-vs-target) queries
- [ ] Write 5 JOIN queries from scratch on paper

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">3</span>
    <span class="day-title">Advanced SQL + Data Validation</span>
    <span class="day-time">⏱️ 3-4 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Study [Subqueries](content/learn.md#subqueries) — IN, EXISTS, correlated
- [ ] Master [Window Functions](content/learn.md#window-functions) — ROW_NUMBER, RANK, LAG, running totals
- [ ] Learn [CTEs](content/learn.md#common-table-expressions-ctes-and-temp-tables)
- [ ] Study [NULL Handling](content/learn.md#null-handling) — critical for ETL!
- [ ] Study [Data Validation Techniques](content/learn.md#data-validation-techniques)
- [ ] Practice [Duplicate Detection](content/sql_practice.md#3-duplicate-detection) and [NULL checking](content/sql_practice.md#4-null-and-blank-checking) queries

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">4</span>
    <span class="day-title">ETL Testing Deep-Dive</span>
    <span class="day-time">⏱️ 3 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Read full [ETL Testing](content/etl_testing.md) section — lifecycle, mapping docs, test design
- [ ] Study [Incremental vs Full Load](content/etl_testing.md#incremental-load-vs-full-load-testing)
- [ ] Review [Common ETL Defect Patterns](content/etl_testing.md#common-etl-defect-patterns) — memorize all 6
- [ ] Study [Banking](content/etl_testing.md#banking-domain-etl-examples) and [Retail](content/etl_testing.md#retail-domain-etl-examples) domain examples
- [ ] Practice [Referential Integrity](content/sql_practice.md#7-referential-integrity-checks) and [Aggregate Comparison](content/sql_practice.md#6-aggregate-comparison-data-reconciliation) queries

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">5</span>
    <span class="day-title">Scenario Practice + Behavioral Prep</span>
    <span class="day-time">⏱️ 3-4 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Work through ALL [Scenario Labs](content/scenarios.md) step by step
- [ ] Study [Interview Tips](content/tips.md) — STAR framework, model answers
- [ ] Read [Resume Guide](content/resume_guide.md) — prepare talking points for every role
- [ ] Practice 5 STAR-format answers out loud
- [ ] Review [Red Flags](content/tips.md#-red-flags--what-not-to-do) to avoid

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">6</span>
    <span class="day-title">Test Yourself</span>
    <span class="day-time">⏱️ 3 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Take the [Timed Quiz](content/quiz.md) — record your score
- [ ] Go through ALL [Flashcards](content/quickrecall.md) — no peeking!
- [ ] Attempt [Mock Interview](mock_interview.md) questions — answer out loud before reading the model answer
- [ ] Review every question you got wrong and re-study those topics
- [ ] If quiz score < 80%, re-study weak areas and retake tomorrow

  </div>
</div>

<div class="plan-day">
  <div class="plan-day-header">
    <span class="day-num">7</span>
    <span class="day-title">Final Review + Confidence</span>
    <span class="day-time">⏱️ 2-3 hours</span>
  </div>
  <div class="plan-day-body">

- [ ] Light review: skim the [Cheat Sheet](content/cheatsheet.md)
- [ ] Quick pass through [Mind Map](content/mindmap.md) — visual reinforcement
- [ ] Re-read your [Resume Guide](content/resume_guide.md) talking points
- [ ] Practice your self-introduction (2 minutes)
- [ ] Review the [Pre-Interview Checklist](content/tips.md#-pre-interview-checklist)
- [ ] Prepare questions to ask the interviewer
- [ ] **Relax and get a good night's sleep** 🌙

  </div>
</div>

> 💡 **Pro Tip:** The most effective study technique is **active recall** — close the material and try to explain concepts from memory. If you can teach it, you know it.
